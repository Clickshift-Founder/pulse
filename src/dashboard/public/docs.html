What is Pulse?

Pulse is a multi-agent DeFi wallet operating system on Solana. Rather than a single bot that executes commands, Pulse is a swarm of specialized autonomous agents that wake up on their own, reason about market conditions, and execute strategies â€” all governed by a Permissioned Brain that makes unsafe trades mathematically impossible.

Core thesis: "The AI cannot harm you even if it hallucinates." Every transaction passes through the Governor's 7-layer safety system before any SOL can move.

Users fund one vault address. The Orchestrator distributes capital, coordinates agents, and sweeps profits to a cold wallet automatically. The user watches it work in real time via the live thought stream.

Heartbeat Architecture

Every agent has a HeartbeatEngine â€” a setInterval loop that fires on a schedule. On each wake cycle, the agent:

  1. Reads HEARTBEAT.md â€” the living directive file (checks for EMERGENCY_STOP)
  2. Reads portfolio state and current prices
  3. Thinks â€” applies strategy logic or AI reasoning
  4. Plans â€” decides what action to take
  5. Executes â€” submits to Governor for approval
  6. Sleeps â€” logs result, waits for next interval
This is what makes Pulse an OS, not a bot. Agents are self-directing â€” they don't need to be called. They wake themselves.

Orchestrator â€” The Vault

The orchestrator's wallet is the vault. Users fund this single address. The orchestrator distributes capital to sub-agents based on their role weights (DCA gets 40%, Trailing gets 25%, etc.). All profits flow back here via the off-ramper before being swept to the cold wallet.

The orchestrator also accepts natural language commands via the Command Centre in the dashboard. Commands like "Change mission to conservative mode" instantly update all agents' directives and are broadcast to the entire swarm.

Governor â€” The Permissioned Brain

The Governor has no wallet â€” it is a decision engine, not an agent. It sits between every AI decision and wallet signing. No transaction can execute without passing all 7 checks:

7 Governor Safety Checks
  1. Balance check â€” sufficient SOL for transaction + gas
  2. Single-tx limit â€” amount â‰¤ GOVERNOR_MAX_SINGLE_TX_SOL (default 0.5 SOL)
  3. Daily spend limit â€” total today â‰¤ GOVERNOR_DAILY_LIMIT_SOL (default 2.0 SOL)
  4. Price impact â€” slippage â‰¤ GOVERNOR_MAX_PRICE_IMPACT_PCT (default 3%)
  5. Liquidity check â€” pool TVL â‰¥ GOVERNOR_MIN_LIQUIDITY_USD (default $50K)
  6. Blacklist check â€” token not in blacklisted set
  7. Rug check â€” RugCheck.xyz score below threshold

The Governor can also recall funds from any agent back to the vault â€” a defensive reversal mechanism. This is the ultimate safety backstop: if an agent misbehaves, its entire allocation can be reclaimed instantly.

Wallet Architecture

Vault Wallet (Orchestrator)

Master treasury. Users fund this address. Orchestrator distributes to sub-agents. Created at first startup and persisted in agent_wallets/orchestrator_main.json.

Agent Wallets (Sub-agents)

Each agent has its own Ed25519 keypair, encrypted with AES-256-GCM using ENCRYPTION_SECRET. Stored in agent_wallets/ â€” NOT in git. The agent can sign transactions autonomously without user interaction.

Cold Wallet (Off-ramp destination)

Your personal external wallet (Phantom, etc.). Set via OFFRAMP_DESTINATION env var. Profits sweep here at the configured threshold (default 15%). Clickbot can convert these to fiat.
